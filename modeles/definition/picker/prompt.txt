Génère un exercice de type "picker" où l'utilisateur doit choisir le terme correct correspondant à une définition donnée, en utilisant les informations du glossaire fournies dans le corps de ma requête.

Ton objectif est de créer un JSON qui suit STRICTEMENT le format ci-dessous.

Le JSON généré doit inclure :

    - title (chaîne de caractères) : Un titre court et pertinent pour l'exercice. Ce titre doit être rédigé dans la langue spécifiée par langueInst et refléter le contenu ou le type de l'exercice (par exemple : "Quiz de Vocabulaire", "Termes à Identifier", "Association Terme/Définition").
    - instructions (chaîne de caractères) : Des instructions claires et conviviales pour l'utilisateur qui fera l'exercice. Ces instructions doivent être rédigées dans la langue spécifiée par langueInst dans les informations supplémentaires du prompt principal. Les instructions doivent clairement énoncer l'objectif de l'exercice (par exemple : "Sélectionnez le terme qui correspond à la définition.", "Trouvez le terme pour la définition suivante."). Il est impératif de NE PAS décrire la méthode d'interaction (par exemple : "Cliquez sur...", "Tapez votre réponse...", "Utilisez le menu déroulant...").
    - questions (liste d'objets) : Cette liste contiendra les questions de l'exercice. Il est impératif que tu génères une question pour CHAQUE terme présent dans le glossaire que je te fournis. Chaque objet dans cette liste doit avoir les clés suivantes :
        - term (chaîne de caractères) : Le terme exact du glossaire auquel cette question fait référence.
        - question_text (chaîne de caractères) : **UNIQUEMENT la définition complète et exacte du terme** tirée du glossaire. C'est la phrase que l'utilisateur doit associer à un terme. La langue de la question doit être celle de la définition du glossaire.
        - options (liste de chaînes de caractères) : Une liste de 3 à 5 options de réponse. **Ces options doivent UNIQUEMENT être les termes** (le terme correct et des distracteurs plausibles tirés d'autres termes du glossaire). Les options ne doivent jamais contenir de phrases de définition.
        - correct_answer (chaîne de caractères) : La chaîne exacte du terme correct parmi les options. Les term et definition utilisés dans les questions et les options DOIVENT être extraits directement et fidèlement des données du glossaire que je te fournis, sans modification ni paraphrase.

Exemple du format JSON attendu (adapte les termes, définitions et options aux données réelles que tu me donnes) :

{
    "title": "Association Terme/Définition IA",
    "instructions": "Sélectionnez le terme qui correspond à la définition.",
    "questions": [
        {
            "term": "Machine Learning",
            "question_text": "A subfield of AI that enables systems to learn from data without being explicitly programmed.",
            "options": [
                "Artificial Intelligence",
                "Machine Learning",
                "Neural Network",
                "Deep Learning"
            ],
            "correct_answer": "Machine Learning"
        },
        {
            "term": "Artificial Intelligence",
            "question_text": "A branch of computer science dealing with the simulation of intelligent behavior in computers.",
            "options": [
                "Artificial Intelligence",
                "Algorithm",
                "Data Science"
            ],
            "correct_answer": "Artificial Intelligence"
        }
    ]
}
